# MyHEBNU — 产品设计文档

> 版本: v1.0 | 创建日期: 2026-06-04 | 状态: 草稿

---

## 1. 产品愿景

**MyHEBNU** 是一款专为河北师范大学学生设计的 Android 教务助手应用。它将学校"新方正教务管理系统"的功能封装在 Material Design 3 风格界面中，让用户在手机上更便捷、更美观地查看课表、成绩和空教室等信息。

### 1.1 核心价值主张
- **更美观** — Material Design 3 现代化 UI，告别老旧的教务系统网页
- **更快捷** — 手机随时查看，无需打开电脑浏览器
- **更智能** — GPA 自动计算、成绩趋势分析、上课/考试提醒

### 1.2 目标用户
- **主用户**: 1 人（管理员本人）
- **扩展性**: 工程结构支持未来多用户扩展

## 2. 用户场景

### 场景 1: 查看课表
> 周日晚上，管理员想快速确认明天的课程安排。打开 APP → 自动进入课表首页 → 看到本周课表，明天的课程高亮显示。

### 场景 2: 查询成绩
> 学期末，管理员想了解本学期的成绩情况。抽屉菜单 → 成绩 → 自动加载本学期成绩 → 查看 GPA 和趋势图表。

### 场景 3: 寻找空教室
> 课间需要找个空教室自习。抽屉菜单 → 空教室 → 选校区和星期几 → 选当前节次 → 查看可用教室列表。

### 场景 4: 确认考试安排
> 考试周前，检查各科考试时间。抽屉菜单 → 考试 → 查看所有考试（距离天数一目了然）。

### 场景 5: 上课提醒
> 上午有早课，APP 在课前 15 分钟推送提醒，告知课程名称和教室。

## 3. 功能需求详述

### 3.1 认证模块
| ID | 需求 | 优先级 |
|----|------|--------|
| A-01 | 支持学校统一身份认证 SSO 登录 | P0 |
| A-02 | 记住登录状态，下次打开自动登录 | P0 |
| A-03 | 会话过期时静默重试登录，失败则提示用户 | P1 |
| A-04 | 支持手动登出，清除所有缓存和凭证 | P1 |

### 3.2 课表模块
| ID | 需求 | 优先级 |
|----|------|--------|
| S-01 | 默认展示**周视图**，横轴周一至周日，纵轴节次 | P0 |
| S-02 | 每门课程卡片显示：课程名、教师、教室、时间、授课周次范围 | P0 |
| S-03 | 不同课程自动分配不同颜色，颜色可记忆 | P0 |
| S-04 | 自动高亮当前周次和当前正在进行的课程 | P0 |
| S-05 | 课表数据本地 Room 缓存，离线可查看 | P0 |
| S-06 | 支持左右滑动切换周次（查看前/后周） | P1 |
| S-07 | 支持下拉手动刷新，从教务系统拉取最新课表 | P1 |
| S-08 | 首次使用时设置当前教学周 | P0 |

### 3.3 成绩模块
| ID | 需求 | 优先级 |
|----|------|--------|
| G-01 | 按学期分类展示全部成绩 | P0 |
| G-02 | 显示课程名称、学分、综合成绩 | P0 |
| G-03 | 点击课程展开**成绩详细构成**（课堂考勤、平时作业、期中考试、期末考试等各子项的百分比权重和得分） | P0 |
| G-04 | 自动计算 GPA / 加权平均分（当前学期 + 累计） | P0 |
| G-05 | 成绩趋势折线图（X轴：学期，Y轴：GPA/均分） | P0 |
| G-06 | 支持不同 GPA 计算方式（4.0/5.0/百分制），可在设置中切换 | P1 |
| G-07 | 数据按需从教务系统拉取，不做本地缓存 | P0 |

> **数据来源**：`/cjcx/cjcx_cxXsKcList.html` 获取学期课程成绩列表，`/cjcx/cjcx_cxXsXmcjList.html` 获取单科成绩明细（含各子项权重和得分，如"课堂表现(20%)=100"、"期末考试(50%)=86"）。详见 [[architecture]] API 文档。

### 3.4 空教室模块
| ID | 需求 | 优先级 |
|----|------|--------|
| R-01 | **必选筛选项**：学年学期、校区 | P0 |
| R-02 | **可选筛选项**：楼号、场地类别、周次、星期几、节次 | P0 |
| R-03 | 查询结果列表展示：教室名称、容量、位置 | P0 |
| R-04 | 数据实时从教务系统拉取，不缓存 | P0 |
| R-05 | 筛选项默认填入当前学期、当前校区、当天 | P1 |

### 3.5 考试安排模块
| ID | 需求 | 优先级 |
|----|------|--------|

| E-02 | 显示距今天数（倒计时） | P1 |
| E-03 | 按时间排序（最近考试在前） | P1 |

### 3.6 通用功能

|----|------|--------|

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| **性能** | 课表加载 < 500ms（缓存命中）；网络请求超时 15s |
| **可靠性** | 网络异常时展示缓存数据 + 错误提示，不崩溃 |
| **安全性** | 登录凭证加密存储（EncryptedSharedPreferences）；HTTPS 通信 |
| **兼容性** | Android 12+ (API 31+) |
| **可维护性** | MVVM + Repository 模式，Hilt 依赖注入，模块化包结构 |
| **国际化** | 中英双语，字符串资源外置 |

## 5. UI/UX 设计规范

### 5.1 设计语言
- **Material Design 3** (Material You)
- 动态取色 (Dynamic Color) 基于用户壁纸
- 圆角卡片、柔和的阴影、流畅的过渡动画

### 5.3 主题
- Light Theme: 浅色背景 + 学校主题色（建议蓝色系）
- Dark Theme: 深色背景 + 低亮度主色
- Dynamic Color: 跟随系统壁纸

### 5.4 交互规范
- **下拉刷新**: Material Design pull-to-refresh 指示器
- **加载状态**: 骨架屏 (Shimmer) 或圆形进度指示器
- **空状态**: 友好的插图 + 说明文字
- **错误状态**: Snackbar 提示 + 重试按钮
- **切换周次**: 左右滑动手势或顶部周次选择器

## 6. 数据流

```
用户操作 → ViewModel (UiState) → Repository
                                     ├── Local DataSource (Room) → 返回缓存
                                     └── Remote DataSource (Retrofit) → 教务系统API
                                          └── 成功 → 更新缓存 → 更新 UiState
                                          └── 失败 → 返回缓存（如有）→ 错误提示
```

## 7. 附录: 交互流程图

### 7.1 首次启动流程
```
启动APP → 检查登录状态
  ├── 已登录/凭证有效 → 自动登录 → 进入课表首页
  └── 未登录 → 显示登录页 → SSO认证 → 设置当前教学周 → 进入课表首页
```

### 7.2 课表刷新流程
```
进入课表页 → 检查本地缓存
  ├── 有缓存 → 先展示缓存 → 后台请求最新数据 → 更新UI
  └── 无缓存 → 显示骨架屏 → 请求数据 → 展示/错误提示
```
